### 启动采集程序（GUI 需要 X 授权）

本机桌面是 **Wayland 会话**，`DISPLAY=:0` 实际是 **Xwayland**，后台进程必须带授权文件，否则报：

```
Authorization required, but no authorization protocol specified
qt.qpa.xcb: could not connect to display :0
```

正确启动方式：

```bash
export DISPLAY=:0
export XAUTHORITY=$(ls /run/user/$(id -u)/.mutter-Xwaylandauth.* | head -1)
/home/luozhisong/miniforge3/envs/rm65/bin/python -u collect_data.py
```

> `cv2` 报 `QFontDatabase: Cannot find font directory ...` 是无害告警，不影响运行。

## 二、采集正确方法（眼在手上 eye-in-hand）

- **标定板固定不动**，相机随机械臂末端一起运动。
- [config.yaml](config.yaml) 的标定板参数必须与实际板子一致：当前 `XX=9, YY=7, L=0.03`（即 10×8 个方格，单格 3cm）。
- **每按一次 `s` 之前，必须先把机械臂移到一个明显不同的姿态**：
  - 相邻两次姿态间**旋转量 ≥ 30°**；
  - 在 **X / Y / Z 三个轴**上都要有旋转变化（例如先绕末端 z 轴转几个角度拍几张，再绕 x 轴、再绕 y 轴）；
  - 始终保证标定板**完整、清晰**出现在相机视野里，且板面与镜头**呈一定角度**（别正对）。
- 拖动示教时**慢一点**（防超速报错），到位后**停稳**，再按一次 `s`。
- 共采集 **15–20 张**，每张一个不同姿态。

> ⚠️ **同一姿态连按多次 `s` 无效**：位姿没变化，对手眼标定（AX=XB）没有信息，混入会触发 `Not enough informative motions` 或得到错误结果。每次 `s` 都要在移动之后。

## 三、常见报错（JSON 协议 `arm_state.err` 字段）

`err` 一旦非 `[0]` 会**持续返回**，必须清除才能继续采集。

| 错误码 | 含义 | 处理 |
|---|---|---|
| **4105** (0x1009) | **超速度限制** | 移动/拖动过快，放慢速度；清错后继续 |
| 4108 (0x100C) | 拖动示教时超速 | 拖动时放慢 |
| 4109 (0x100D) | 机械臂发生碰撞 | 排查碰撞，清错 |
| 4116 (0x1014) | 超关节软限位 | 长按末端绿色按钮恢复到限位内 |
| 4099 (0x1003) | 奇异点 | 规避奇异姿态 |
| 4098 (0x1002) | 目标角度超限位 | 修改下发指令 |

**清除报错**（任选）：
- JSON 命令：`{"command": "clear_arm_err"}`（代码已集成到采集程序的自动重试中）；
- 浏览器打开 Web 示教器 `http://192.168.1.18`，点清除报错。

完整错误码表：<https://develop.realman-robotics.com/robot/json/errorList/>

## 四、collect_data.py 代码改动（2026-06-29）

原代码在按 `s` 采集失败时存在两个问题，已修复：

1. **跳号 bug**：原 `count += 1` 位于 `if state:` 之外，采集失败（机械臂报错）时 `count` 仍自增，导致下一次成功保存的图片编号跳号，与 `poses.txt` 行数错位 → 标定数据错乱。**现改为仅在成功保存后才自增。**
2. **自动清错 + 重试**：按 `s` 获取位姿失败时，自动发送 `clear_arm_err` 清除机械臂报错并重试一次，保证图片与位姿始终 1:1 对应。

> 动作：拖动示教移动到位 → 停稳 → 按 `s`。即便偶发超速/碰撞，程序会自动清错重试，无需手动干预。

## 五、整体流程

1. **采集**：运行 `collect_data.py`，数据存入 `eye_hand_data/dataYYYYMMDDNN/`（`N.jpg` 图片 + `poses.txt` 位姿）。
2. **计算**：
   - 眼在手上：`compute.py`（config.yaml 设 `calib_mode: eye_in_hand`）→ 相机坐标系相对于**机械臂末端**坐标系的 R、t；
   - 眼在手外：`compute.py`（config.yaml 设 `calib_mode: eye_to_hand`）→ 相机坐标系相对于**机械臂基座**坐标系的 R、t。
3. **结果**：旋转矩阵 R、平移向量 t（单位：米）、四元数。

> 误差范围：平移向量与实际差距约 1cm 内（受图片质量影响）。
