# RM65 双 RealSense D435 相机标定方案

> 系统：RealMan **RM65-B** 六轴机械臂 + 两路 Intel RealSense D435
> - 相机 A（SN `261722075459`）：安装于机械臂末端法兰 → **眼在手上 (eye-in-hand)**
> - 相机 B（SN `261722076078`）：固定于机械臂正前方、俯视 → **眼在手外 (eye-to-hand)**
> 目标：把两路相机的测量统一到机械臂 **基座坐标系 (base_link)**。

---

## 硬件参数 (RM65-B)

机械臂为 **RM65-B（标准版）**，参数取自官方[本体参数](https://develop.realman-robotics.com/robot4th/robotParameter/RM65OntologyParameters/)。与标定强相关项：

| 参数 | 值 | 对标定的影响 |
|---|---|---|
| 构型 / 自由度 | 仿人构型 / 6 DOF | — |
| 工作半径 | 610 mm（球体，**剔除基座正上方/正下方圆柱区**） | 限定标定板 / 相机布置范围 |
| 重复定位精度 | **±0.05 mm** | 机器人位姿噪声 ≪ 相机深度误差，**残差主要由相机侧决定**（见 §7） |
| 有效负载 | 5 kg | 携带相机 A + 标定板无压力 |
| 末端法兰 d6 | **144 mm**（RM65-B，MDH 第 6 轴 di） | SDK 上报的 TCP 原点需对齐该法兰面（见 §1） |
| 关节范围 | J1±178 / J2±130 / J3±135 / J4±178 / J5±128 / J6±360° | 规划采集位姿工作包络 |

> 位姿规划务必规避 RM65-B 的四类奇异点（落入即样本失效）：**肩部奇异**、**肘部奇异**（`q3=0`）、**腕部奇异**（`q5=0`）、**边界奇异**（`q3=0 且 q5=0`）。详见 §2 采集规范。

## 0. 标定目标（要解什么）

最终需要得到三组参数，并据此把任意相机像素/点云映射到基座系：

| 序号 | 标定项 | 待求量 | 含义 | 用于 |
|---|---|---|---|---|
| ① | 相机内参 | `K, dist`（每路） | 焦距/主点/畸变 | solvePnP、去畸变 |
| ② | 手眼外参（相机 A） | `T_F_A` | 相机 A 光心系 → 末端法兰系 | 眼在手上 |
| ③ | 眼在手外外参（相机 B） | `T_B_E` | 相机 B 光心系 → 基座系 | 眼在手外 |

> 记号约定：`T_X_Y` 表示“把 Y 系下的点变到 X 系”的 4×4 齐次矩阵，即 `P_X = T_X_Y · P_Y`。
> 下文 F=末端法兰(link6/TCP)、A=相机A、E=相机B、B=基座、Tg=标定板。

统一映射公式（标定成功后即可实时用）：
- 相机 A（随臂动）：`P_B = T_B_F(t) · T_F_A · P_A`，其中 `T_B_F(t)` 为实时的机器人正运动学位姿。
- 相机 B（固定）：`P_B = T_B_E · P_E`。

两路相机由此共享同一基座系，可互相校验、可融合。

## 1. 坐标系定义（务必先统一约定）

| 坐标系 | 原点 / 朝向 | 来源 |
|---|---|---|
| base (B) | 机械臂底座，Z 向上 | RM65 SDK 正运动学输出基准 |
| flange (F) | 末端法兰盘中心 | RM65 SDK TCP 位姿（注意减去夹爪/工具偏置，统一到法兰） |
| 相机光心系 (A/E) | OpenCV/Realsense 光心约定：**Z 向前、X 向右、Y 向下** | pyrealsense2 |
| 标定板系 (Tg) | 棋盘/CharUco 左上角内角点，Z 出板面 | 自定义 |

⚠️ 两类约定差异最大、最易出错的地方：
1. **相机光心系（Z 前）与机器人系（Z 上）不一致**，做手眼解算前不要手动“纠正”方向——`cv2.calibrateHandEye` 会自动吸收这部分旋转，前提是输入位姿自洽。
2. **末端 TCP**：若夹爪/工具末端有偏置，先在 RM65 SDK 中把 TCP 设回**法兰中心**再记录位姿，否则 `T_F_A` 会混入工具偏置。（RM65-B 末端法兰 d6 = 144 mm，确认 SDK 上报的 TCP 原点落在该法兰面中心。）
3. **旋转表示**：RM65 位姿可能是 RPY / 四元数 / 轴角，统一转成 3×3 旋转矩阵 + 平移向量再喂给 OpenCV。

## 2. 准备工作

### 2.1 标定板（强烈推荐 CharUco）
- 推荐 **CharUco 板**：兼具棋盘精度与 ArUco 鲁棒性，部分遮挡/部分出画面仍可检测，适合机械臂自动采集。
- 备选：经典棋盘格（`calibrateCamera` 友好，但对位姿要求严）。
- 关键参数必须**实测并写死**：方格物理边长 `square_len`、ArUco 标记边长 `marker_len`、行列数、字典。打印后用游标卡尺复核（误差 < 0.1 mm），并平整裱在硬板/铝板上，禁止翘曲。

### 2.2 环境
- 光照均匀、无强反光、无运动模糊；D435 RGB 与 IR 投射器不打架（必要时盖住 IR 投射器，仅用 RGB 内参）。
- 相机 A 工作距离建议 **0.30–0.80 m**（D435 最小深度默认 ~0.3 m）；相机 B（俯视）覆盖机械臂工作台中心。

### 2.3 软件
- `pyrealsense2`（取流、读取出厂内参、对齐深度到 RGB）
- `opencv-python`（`findCharucoBoard` / `solvePnP` / `calibrateHandEye`）
- `numpy` / `scipy.spatial.transform`（RPY↔矩阵）
- 可选：ROS `easy_handeye` + `aruco_ros`（若走 ROS 生态）。本方案默认 **纯 Python 独立实现**，便于嵌入 RM65Tools。

### 2.4 数据采集通用规范（影响成败）
- 每个标定环节采集 **N ≥ 15–20 组**（位姿, 图像）对。
- 位姿**多样性**：平移覆盖工作空间、姿态需绕 **≥2 个不平行轴**有显著旋转（纯单轴旋转 / 纯平移会使 AX=XB 退化，解不准）。
- **规避奇异点（RM65-B）**：采集位姿须远离四类奇异——肩部奇异（关节5/6 轴线交点落在“过 J1 轴且平行 J2 轴的平面”）、肘部奇异（`q3=0`）、腕部奇异（`q5=0`）、边界奇异（`q3=0 且 q5=0`）。建议 `|q3|>20°`、`|q5|>20°`，并避免末端伸到最远端。
- **工作空间（RM65-B）**：半径 610 mm 的球体（**剔除基座正上方/正下方的圆柱区**），标定板布置与相机 B 俯视覆盖均需落在该球体内；可操作度较好的是中段环状区域。
- 每张图：标定板占视野较大比例、角点清晰、对焦清晰、**无运动模糊**（臂停稳后再触发相机）。
- 同步：先 `moveJ` 到位 → 等 0.5–1 s 稳定 → 读机器人位姿 → 触发一帧 RGB（避免位姿与图像错位）。

## 3. 相机内参（① 确认 / 复校）

D435 出厂已带 RGB/深度内参，可直接从数据流读取：

```python
profile = pipeline.get_active_profile()
intr = profile.get_stream(rs.stream.color).as_video_stream_profile().get_intrinsics()
# intr.fx, intr.fy, intr.ppx, intr.ppy, intr.coeffs (k1,k2,p1,p2,k3), intr.model
```

- **手眼标定阶段**：直接用出厂内参做 `solvePnP` 即可（出厂精度对手眼足够）。
- **若需高精度**：用 CharUco/棋盘采 30–50 张图，`cv2.aruco.calibrateCameraCharuco` 复校 RGB 内参与畸变；深度可用 `rs2` 的 `rs-camera` / `rs-depth` 工具评估。
- 内参结果写入存档（见 §6），并标注是“出厂”还是“复校”。

## 4. 相机 A（5459，眼在手上）手眼标定（②）

### 4.1 布置
- 标定板 **固定**在桌面（世界/基座系下不动），板面朝向机械臂末端相机。
- 相机 A 随臂运动，整个过程标定板始终在相机视野内。

### 4.2 采集（每个位姿 k）
1. 控臂到第 k 个位姿，停稳。
2. 读机器人 **法兰**位姿 → `T_B_F(k)`（4×4）。
3. 采一帧 RGB，检测 CharUco，`cv2.solvePnP` 得到 **板在相机系**的位姿 `T_A_Tg(k)`。
4. 记录三元组 `(k, T_B_F(k), T_A_Tg(k))`。

### 4.3 解算（AX = XB）
- 模型：`A_ij · T_F_A = T_F_A · B_ij`
  - `A_ij = T_B_F(j)⁻¹ · T_B_F(i)`（末端相对运动）
  - `B_ij = T_A_Tg(j) · T_A_Tg(i)⁻¹`（板在相机系中的相对运动）
  - `X = T_F_A`（待求）
- OpenCV 调用：

```python
R_grip2base = [T_B_F[k][:3,:3]  for k]   # 每个位姿的旋转
t_grip2base = [T_B_F[k][:3, 3]  for k]   # 平移
R_tgt2cam   = [T_A_Tg[k][:3,:3] for k]
t_tgt2cam   = [T_A_Tg[k][:3, 3] for k]
R_F_A, t_F_A = cv2.calibrateHandEye(
    R_grip2base, t_grip2base, R_tgt2cam, t_tgt2cam,
    method=cv2.CALIB_HAND_EYE_PARK)        # PARK / DANIILIDIS 精度更稳
T_F_A = np.eye(4); T_F_A[:3,:3]=R_F_A; T_F_A[:3,3]=t_F_A.ravel()
```

- **建议**：用 `TSAI / PARK / DANIILIDIS` 三种方法各解一遍，取旋转/平移的中位数或一致性最好的；三者差异大说明采集位姿退化，需补采。

## 5. 相机 B（6078，眼在手外）标定（③）

### 5.1 布置
- 相机 B 固定俯视不动。
- 标定板 **随机械臂运动**：把板可靠地固定/夹在末端（板相对法兰不动，即 `T_F_Tg` 恒定）。

### 5.2 采集（每个位姿 k）
1. 控臂带动标定板到第 k 个位姿，停稳，板面正对相机 B。
2. 读法兰位姿 `T_B_F(k)`。
3. 采一帧，`solvePnP` 得 `T_E_Tg(k)`（板在相机 B 系下位姿）。
4. 记录 `(k, T_B_F(k), T_E_Tg(k))`。

### 5.3 解算（eye-to-hand）
- 待求 `T_B_E`（相机 B 在基座系）。OpenCV 的标准做法是**把机器人位姿取逆**后当作 eye-in-hand 输入：

```python
# 关键：eye-to-hand 传入的是 base←gripper（取逆）
R_base2grip = [T_B_F[k][:3,:3].T for k]    # 旋转取逆
t_base2grip = [-T_B_F[k][:3,:3].T @ T_B_F[k][:3,3] for k]
R_tgt2cam   = [T_E_Tg[k][:3,:3] for k]
t_tgt2cam   = [T_E_Tg[k][:3, 3] for k]
R_B_E, t_B_E = cv2.calibrateHandEye(
    R_base2grip, t_base2grip, R_tgt2cam, t_tgt2cam,
    method=cv2.CALIB_HAND_EYE_PARK)
T_B_E = np.eye(4); T_B_E[:3,:3]=R_B_E; T_B_E[:3,3]=t_B_E.ravel()
```

- ⚠️ OpenCV 输出符号随版本/约定易混；解算后 **务必用 §7 验证**（投影一个已知点到基座系），符号反了会立刻暴露。

## 7. 联合验证与验收标准

> 误差预算：RM65-B 重复定位精度 **±0.05 mm**，远小于 D435 深度误差（≈2%×距离）。下述残差阈值主要由**相机 / 深度侧**决定；若残差系统性偏大且方向固定，优先排查坐标系约定与符号（见 §1），而非机器人精度。

### 7.1 内部级联验证（无需额外设备）
1. **重投影误差**：每组 `solvePnP` 的角点重投影误差应 **< 1 px**（理想 < 0.5 px）。
2. **AX=XB 一致性残差**：用求得的 `X` 把所有 `A_ij`、`B_ij` 对齐，旋转残差 **< 1°**、平移残差 **< 2–3 mm**（视工作距离）。
3. **三方法一致性**：TSAI/PARK/DANIILIDIS 结果旋转差 **< 1°**、平移差 **< 2 mm**。

### 7.2 跨相机统一验证（两路同框）
- 把一个 ArUco 标记固定在工作台某处，使 **A、B 同时可见**（或在相邻位姿分别观测）：
  - 经 A 映射到基座：`P_B^A = T_B_F · T_F_A · P_A`
  - 经 B 映射到基座：`P_B^B = T_B_E · P_E`
  - 两路算出的同一点位差应 **< 3–5 mm**（D435 深度误差 ~2%×距离，主导该误差）。

### 7.3 绝对验证（物理触点）
- 用末端（或固定的针尖）去触一个已知物理点（该点坐标可由相机 B 测得并映射到基座），比对触点时机器人正运动学给出的基座坐标与相机估计坐标，残差应 **< 3–5 mm**。这是最能暴露符号/约定错误的检验。

### 7.4 深度链路验证
- 在标定板/标记角点上，用对齐后的深度 + 内参反算 3D 坐标，再走外参映射，与上述结果比对，确认 `rs.align(depth→color)` 配置正确、深度尺度（米）正确。

**验收门槛（建议）**：内参重投影 < 1 px；手眼残差平移 < 3 mm、旋转 < 1°；两路跨相机一致 < 5 mm；物理触点验证 < 5 mm。达不到则按 §8 排查。

## 8. 采集规范与避坑清单

| 症状 | 原因 / 处理 |
|---|---|
| 三方法结果差异巨大 | 位姿退化（缺轴向旋转）。补采绕不同轴的旋转位姿。 |
| 手眼平移偏大、不稳定 | 平移变化不足；板距离太近/太远；运动模糊。增加工作距离上的多样性，停稳再采。 |
| 跨相机验证系统性偏移（固定方向） | 多半是**约定/符号错误**：TCP 未回到法兰、RPY 顺序/坐标轴理解错、eye-to-hand 未取逆。逐项核对 §1。 |
| 深度映射整体偏 | `align` 未开、深度单位非米、或左右相机基线/深度对齐未启用。检查 `rs.align`。 |
| 偶发大残差个别样本 | 图像模糊/遮挡/反光。剔除该样本重解。 |
| 重投影误差整体偏大 | 内参不准或板物理尺寸填错。复核 `square_len/marker_len` 实测值。 |

通用准则：
- 先验内参 → 再手眼；眼在手上、眼在手外**分开采、分开解**，互不混用样本。
- 全程记录原始 `(位姿, 图像)`，便于复算与排错。
- 标定环境固定后，相机/标定板**不可再动**，动过则需重标。

## 9. 执行顺序速查

1. 装标定板，确认物理尺寸实测值、TCP 回到法兰、相机取流与内参读取正常。
2. **相机 A（眼在手上）**：板固定 → 采 ≥18 组 `(T_B_F, RGB)` → 内参出厂直用 → `solvePnP` → `calibrateHandEye(PARK)` → 存 `T_F_A`。
3. **相机 B（眼在手外）**：板随臂 → 采 ≥18 组 `(T_B_F, RGB)` → `solvePnP` → 机器人位姿取逆 → `calibrateHandEye(PARK)` → 存 `T_B_E`。
4. **验证**：重投影 / 一致性残差 / 跨相机同框 / 物理触点，逐项达标。
5. **存档**：写入 `cameras/<SN>/*.json`，记录方法、残差、时间戳。

完成后即可用统一公式把两路 D435 的测量实时映射到 RM65 基座系。
